# Нахождение  гамильтонова  обхода  в  неориентированном   графе

Сейчас уже всего не упомню, поэтому, буду объяснять в общих чертах.
В общем, у нас есть граф, который представляется в виде вершин и графов.
Он хранится в памяти в упорядоченном виде, в двумерном массиве.
Типа, каждая вершина с каждой, и в ячейке отмечено, есть ли между ними связь.
Короче, из этого можно получить упорядоченный список рёбер.
Решения задачи - это последовательность рёбер, по которым происходит обход.
То есть, набор 0 и 1. Накладываем их на рёбра, и обходим по ним граф.
То есть, если в графе 5 рёбер, например, то решением будет последовательность типа 11100.
Ну и там где-то в проге есть функция приспособленности.
Она проверяет, является ли цикл гамильтоновым и считает, насколько он к нему близок.
При этом, если последовательность вообще не замкнута, то она нещадно штрафуется.
